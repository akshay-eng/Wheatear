Your agents shouldn't be grounded on one platform.

Wheatear is a migration engine for AI agents. It connects to Copilot Studio or watsonx Orchestrate with your credentials, discovers every agent (tools, knowledge, collaborators) and moves the whole workflow to the other platform. Deterministic where correctness matters. AI only as a consented last mile.

Switching platforms means rebuilding everything by hand. That's not a migration — that's lock-in doing its job.

Every orchestration platform speaks its own dialect. The moment your agents matter, the cost of leaving becomes the reason you can't. Three gaps do most of the damage:

The connector gap

1,400+ connectors vs. bring-your-own

Copilot Studio ships prebuilt connectors for ServiceNow, Jira, Salesforce, SAP. Orchestrate expects MCP servers and OpenAPI specs. Your agent's tools don't translate by copy-paste. The exception: tools that were MCP all along. Wheatear detects those and re-points the endpoint: bridge: native_mcp, zero rebuild. Everything else becomes a named, actionable task, never a silent loss.

The format gap

Dataverse solution XML vs. ADK YAML

One side exports a zip of Dataverse XML: topics, GPT components, bot manifests. The other wants agent.yaml with instructions, guidelines, and collaborator lists. Wheatear parses both real export shapes (calibrated against actual production exports, not documentation) and round-trips them through one canonical IR.

The knowledge gap

Live SaaS indexes vs. vector stores

A SharePoint knowledge source has no equivalent on a platform that expects Milvus or Elasticsearch. Pretending otherwise ships a broken agent. Wheatear emits a re-ingestion plan per source: reindex_vector, upload, or custom_service — so a human closes the gap deliberately, once.

Deterministic core

NO MODEL · AUDITABLE · REPEATABLE

Everything touching schemas, credentials, endpoints, or structure is pure field mapping. Run it twice, get the same bytes.

  • Export parsing: both real Copilot shapes, three Orchestrate shapes
  • Instructions carried over verbatim, lossless for generative agents
  • MCP endpoints re-pointed, models tier-mapped, guidelines restructured
  • Solution XML / agent YAML generation + deploy

AI last mile — opt-in

ASKS FIRST · NEVER LOAD-BEARING

The model is reserved for judgment calls, and it asks before touching anything:

  • Adapting prose instructions to the target platform's idiom
  • Repairing a rejected import: you see the error, you approve the fix
  • Decomposing dialog-tree logic into structured guidelines
